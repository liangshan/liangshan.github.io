安居客

资深开发工程师 2011/03—今 在安居客到目前为止基本分为 3 个阶段。

好租工程师阶段，在租房事业部做业务项目的开发。期间带动了好租的技术学习氛围，对好租的多个模块都有改造，和需求方的关系也处理的很好。所以很快就接触到了公司核心的工程师，参与了集团规模的几个项目。比较重要的是搜索流程的改造以及 SVN 到 Git 的切换，以及安居客私有的 RPC 协议。

搜索流程改造是指将用户的字符串搜索，转化为试图去猜测用户意图。先用朴素贝叶斯来判断搜索词与地理信息的关系，然后去 POI 库搜寻相应位置的坐标，最终将距离维度加入了排序规则。该流程已经申请专利「垂直搜索引擎的实现方法」，可在国家专利局搜索到。

Git 的切换是指将代码版本控制工具从 SVN 切换到 Git。这需要制定流程，以及准备相应的持续集成的工具。

安居客私有 RPC 协议是指，基于 ØMQ 的消息协议。我主要负责将这套协议用于生产。

第二个阶段，在二手房事业部做工具开发。期间主要完成了几件事情，一个是设计了如何针对安居客 PHP 框架中代码的执行速度做可视化。第二个是做安居客技术委员会的「秘书」。第三个是开始做推荐引擎。

PHP 代码执行速度可视化，基本思路是在框架底层加入统计支持，在进程结束后通过 RPC 协议发送给服务端，服务端做出汇总计算并提供 Web 界面来绘图。

公司运行了几年，但技术部的一些知识没有很好的沉淀。我主要负责推进技术部第一次大规模的文档整理工作。

我们意识到需要使用开源工具做推荐引擎，基于 Apache Mahout 的推荐引擎第一版 ROMAR 完成，我作为项目组的组长带领不同团队的工程师完成这个项目。

第三个阶段，在总部架构部做工具开发。主要负责集团性的基础服务及工具开发和维护。期间主要完成了几个项目：推荐引擎的持续推进，安居客私有的类 GitHub 项目。

制定了未来公司推荐的发展方向，得到高层支持并推广至集团层面。设计中考虑了引擎的升级及配套的运维、数据收集的工具。

在当时没有很好开源项目选择的情况下，我们决定自己开发类 GitHub 的项目。实现了 issue, pull-request, organization 等核心的功能，并在过程中总结了安居客 Web 开发的最佳实践。

整体来说，从一个工程师到公司的资深工程师，在安居客的过程中，逐步的加深了对技术的理解。同时也开始通过阅读来将实践中得到的经验理论化，逐渐形成了较宽知识面。
